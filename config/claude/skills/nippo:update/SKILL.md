# タスク更新 (nippo:update)

`~/.claude/nippo/tasks.md` のタスクを変更します。追加以外の操作（完了・修正・削除・移動）に対応します。

## 手順

1. `~/.claude/nippo/tasks.md` を読み込む
2. 引数から変更内容を解釈する。引数がない場合はユーザーに確認する
3. 変更種別を判断し、対応する操作を実行する
4. ファイルを更新する
5. 変更内容をユーザーに伝える

## 変更種別と操作

### 完了（タスクを未完了→完了に移動）

「完了」「done」「終わった」などの表現で判断する。

- 対象タスクを「未完了タスク」セクションから削除する
- 「完了タスク」セクションに `| 完了: {YYYY-MM-DD}` を末尾に追加して移動する
- 作業ログに完了エントリを追記する（既存の当日ログがあれば「完了タスク」に追記、なければ新規ログを作成する）

### 修正（タスク内容の変更）

「変更」「修正」「更新」「rename」などの表現で判断する。

- 対象タスクの内容を新しい内容に書き換える
- 追加日は保持したまま変更する

### 削除（タスクの除去）

「削除」「remove」「消す」などの表現で判断する。

- 対象タスクを該当セクションから削除する

### 未完了に戻す（完了→未完了）

「差し戻し」「reopen」「未完了に戻す」などの表現で判断する。

- 対象タスクを「完了タスク」セクションから削除する
- 「未完了タスク」セクションに `| 完了:` 以降を除いた形式で移動する

## tasks.md のフォーマット

未完了タスクの行形式：

```
- {タスク名} | プロジェクト: {プロジェクト名} | 追加: {YYYY-MM-DD}
```

完了タスクの行形式：

```
- {タスク名} | プロジェクト: {プロジェクト名} | 追加: {YYYY-MM-DD} | 完了: {YYYY-MM-DD}
```

## タスクの特定

引数からタスク名のキーワードで部分一致検索する。複数候補がある場合はユーザーに選択させる。

## 応答

変更後、以下を表示する：
- 実行した操作の種別と対象タスク
- 現在の未完了タスク数

例（完了の場合）：
```
✅ タスクを完了しました：
  Spec差分更新PRのマージ・他セクションへの横展開 | プロジェクト: ADeT-AI | 完了: 2026-03-26

📋 未完了タスク: 0件
📁 ~/.claude/nippo/tasks.md
```
