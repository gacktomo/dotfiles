# タスク更新 (nippo:update)

`~/.claude/nippo/tasks.md` のタスクを変更します。追加以外の操作（完了・修正・削除）に対応します。

## 手順

1. `~/.claude/nippo/tasks.md` を読み込む
2. 引数から変更内容を解釈する。引数がない場合はユーザーに確認する
3. 変更種別を判断し、対応する操作を実行する
4. `tasks.md` を更新する
5. 完了操作の場合、夕日報ファイルに作業メモを追記する（後述）
6. 変更内容をユーザーに伝える

## 変更種別と操作

### 完了（タスクを削除）

「完了」「done」「終わった」などの表現で判断する。

- 対象タスクを「未完了タスク」セクションから削除する
- 夕日報ファイル（`~/.claude/nippo/reports/YYYY-MM-DD-evening.md`）に作業メモを追記する

### 修正（タスク内容の変更）

「変更」「修正」「更新」「rename」などの表現で判断する。

- 対象タスクの内容を新しい内容に書き換える
- 追加日は保持したまま変更する

### 削除（タスクの除去）

「削除」「remove」「消す」などの表現で判断する。

- 対象タスクを未完了タスクセクションから削除する

## tasks.md のフォーマット

未完了タスクの行形式：

```
- {タスク名} | プロジェクト: {プロジェクト名} | 追加: {YYYY-MM-DD}
```

## タスクの特定

引数からタスク名のキーワードで部分一致検索する。複数候補がある場合はユーザーに選択させる。

## 夕日報ファイルへの作業メモ追記（完了時のみ）

完了操作を実行したとき、`~/.claude/nippo/reports/YYYY-MM-DD-evening.md` に完了タスクを作業メモとして記録する。

### ファイルが存在しない場合

以下の初期構造で新規作成する：

```markdown
# 作業メモ

## 完了タスク

- {タスク名} | プロジェクト: {プロジェクト名} | 完了: {YYYY-MM-DD HH:MM}
```

### ファイルが存在する場合

- `## 完了タスク` セクションがあれば、その末尾に追記する
- セクションがなければ、ファイル末尾に `## 完了タスク` セクションを追加して追記する

追記する行の形式：

```
- {タスク名} | プロジェクト: {プロジェクト名} | 完了: {YYYY-MM-DD HH:MM}
```

## 応答

変更後、以下を表示する：
- 実行した操作の種別と対象タスク
- 現在の未完了タスク数
- 完了操作の場合、夕日報ファイルへの追記パス

例（完了の場合）：
```
✅ タスクを完了しました：
  Spec差分更新PRのマージ・他セクションへの横展開 | プロジェクト: ADeT-AI

📋 未完了タスク: 0件
📝 作業メモを追記: ~/.claude/nippo/reports/2026-03-26-evening.md
```
