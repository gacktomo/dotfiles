# 作業進捗メモ更新 (nippo:update)

当日の朝日報（`~/.claude/nippo/reports/YYYY-MM-DD-morning.md`）の `■ 作業進捗メモ` セクションを更新します。

## 手順

1. 今日の日付（YYYY-MM-DD）を取得する
2. `~/.claude/nippo/reports/YYYY-MM-DD-morning.md` を読み込む
3. ファイルが存在しない場合、朝日報がまだ作成されていない旨を伝えて終了する
4. `■ 今日やること` と `■ 作業進捗メモ` の現在の内容を表示し、ユーザーに以下を問う：
   - どのタスクの進捗を記録するか（番号で選択）
   - または新規メモとして自由記述するか
   - 引数がある場合はその内容を確認として提示し、どのタスクに紐づけるか問う
5. ユーザーの回答に基づいてメモ内容を確定する
6. `■ 作業進捗メモ` セクションを更新して保存する
7. 更新内容をユーザーに伝える

## ユーザーへの問い（ステップ4の表示例）

```
📋 本日のタスク
1. screen detail, feature PRのマージ
2. databaseの差分Spec生成対応（APIもできれば）

■ 作業進捗メモ（現在）
- （まだメモなし）

どのタスクを更新しますか？
[1] タスク1: screen detail, feature PRのマージ
[2] タスク2: databaseの差分Spec生成対応
[N] 新規メモを追記
```

## 作業進捗メモの更新ルール

- `■ 作業進捗メモ` セクションが存在しない場合は、`■ 完了ライン` セクションの直前に追加する
- メモは箇条書き形式で追記する（既存のメモを消さずに末尾に追記）
- 各メモ行のフォーマット：`- {内容} | {HH:MM}`
- 現在時刻を HH:MM 形式で付与する

## 朝日報のフォーマット（メモセクション追加後）

```markdown
🌅 開始
【本日の予定】

■ 今日やること
1. ...

■ 作業進捗メモ
- {メモ内容} | {HH:MM}

■ 完了ライン
1. ...
```

## 応答

```
✅ 作業進捗メモを更新しました

追記内容: {追記したメモ}

📁 ~/.claude/nippo/reports/YYYY-MM-DD-morning.md
```
