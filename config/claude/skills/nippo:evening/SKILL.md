# 夕方の日報生成 (nippo:evening)

当日の朝日報の `■ 作業進捗メモ` セクションを参照して、本日の夕日報を生成します。

## 手順

1. 今日の日付（YYYY-MM-DD）を取得する
2. `~/.claude/nippo/reports/YYYY-MM-DD-morning.md`（今日）を読み込む
3. 朝日報の `■ 今日やること` セクションと `■ 作業進捗メモ` セクションを抽出する
4. 朝日報が存在しない場合は、その旨を伝えて終了する
5. 夕日報マークダウンを生成する
6. `~/.claude/nippo/reports/YYYY-MM-DD-evening.md` に保存する
7. チャットにマークダウン内容を表示する

## 夕日報のフォーマット

```markdown
🌙 終了（夕方）
【本日の実績】

■ サマリ
- 進捗：XX%
- 状態：順調 / やや遅延 / 遅延

■ 進捗詳細
1. ○○：100%
2. ○○：60%
3. ○○：未着手

■ 定量
- 完了タスク：X / Y
- 想定との差分：+Xh / -Xh (工数時間の状況をプラスかマイナスで表現)

■ 課題・リスク
- （例：○○で詰まり、明日以降に影響の可能性あり）

■ 対応方針
- （どう解決するか / 相談するか）

■ 明日の予定
1. ○○
2. ○○
```

## 各セクションの埋め方

- **サマリ**：作業進捗メモから完了・進行中タスクを把握し進捗率を算出。状態は進捗率を目安に判定（80%以上→順調、50〜79%→やや遅延、50%未満→遅延）
- **進捗詳細**：朝日報の「今日やること」を基準に、「作業進捗メモ」の内容から各タスクの進捗率を推定して列挙
- **定量**：作業進捗メモで完了と明記されたタスク数を完了タスクとして計上。朝日報の予定タスク数と比較
- **課題・リスク**：作業進捗メモで詰まりや問題が記録されていれば記載。なければ「なし」
- **対応方針**：課題に対する次のアクション。課題がなければ省略
- **明日の予定**：未完了タスクと新規タスクを優先度順に列挙

## 応答

チャットには以下を表示する：
1. 生成した夕日報の全文（マークダウンレンダリング）
2. 保存先パス

例：
```
📊 夕方の日報を生成しました

[日報内容]

💾 保存先: ~/.claude/nippo/reports/2026-03-18-evening.md
```
