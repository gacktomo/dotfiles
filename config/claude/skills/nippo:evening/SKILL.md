# 夕方の日報生成 (nippo:evening)

`~/.claude/nippo/tasks.md` の残タスクと夕日報ファイルの作業メモを元に、本日の夕日報を生成します。

## 手順

1. `~/.claude/nippo/tasks.md` を読み込む
2. `~/.claude/nippo/reports/YYYY-MM-DD-evening.md` が存在すれば読み込む（作業メモ・完了タスク記録として使用）
3. `~/.claude/nippo/reports/YYYY-MM-DD-morning.md` が存在すれば読み込む（朝の目標との比較用）
4. 今日の日付（YYYY-MM-DD）を取得する
5. 夕日報マークダウンを生成する
6. `~/.claude/nippo/reports/YYYY-MM-DD-evening.md` に保存する（既存の作業メモは上書きせず、日報セクションを追記・統合する）
7. チャットにマークダウン内容を表示する

## 夕日報のフォーマット

```markdown
🌙 終了（夕方）
【本日の実績】

■ サマリ
- 進捗：XX%
- 状態：順調 / やや遅延 / 遅延

■ 進捗詳細
1. ○○：100%
2. ○○：60%
3. ○○：未着手

■ 定量
- 完了タスク：X / Y
- 想定との差分：+Xh / -Xh (工数時間の状況をプラスかマイナスで表現)

■ 課題・リスク
- （例：○○で詰まり、明日以降に影響の可能性あり）

■ 対応方針
- （どう解決するか / 相談するか）

■ 明日の予定
1. ○○
2. ○○
```

## 各セクションの埋め方

- **サマリ**：完了タスク数÷予定タスク数で進捗率を算出。状態は進捗率を目安に判定（80%以上→順調、50〜79%→やや遅延、50%未満→遅延）
- **進捗詳細**：夕日報ファイルの「完了タスク」セクションと tasks.md の残タスクを合わせて列挙。完了は100%、残タスクは0%、作業中は会話ログから推定
- **定量**：完了タスク数は夕日報ファイルの「完了タスク」セクションから。想定差分は朝日報の予定と実績の比較（朝日報がない場合は省略）
- **課題・リスク**：会話ログで詰まった箇所・未解決の問題を記載。なければ「なし」
- **対応方針**：課題に対する次のアクション。課題がなければ省略
- **明日の予定**：残タスクと新規タスクを優先度順に列挙

## 応答

チャットには以下を表示する：
1. 生成した夕日報の全文（マークダウンレンダリング）
2. 保存先パス

例：
```
📊 夕方の日報を生成しました

[日報内容]

💾 保存先: ~/.claude/nippo/reports/2026-03-18-evening.md
```
