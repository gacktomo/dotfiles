# PR作成スキル (pr-create)

現在のブランチとmainブランチの差分を分析し、GitHub Pull Requestを作成する。

## 手順

### 1. 事前確認

```bash
# 現在のブランチを確認
git branch --show-current

# リモートの最新状態を確認
git fetch origin

# mainとの差分コミットを確認
git log origin/main..HEAD --oneline

# ファイルの変更差分を確認
git diff origin/main..HEAD --stat

# 詳細なdiffを取得（内容分析のため）
git diff origin/main..HEAD
```

未コミットの変更がある場合はユーザーに確認してから進む。

### 2. 差分の分析

diff情報から以下を抽出・分析する：

- **変更の種類**: 新機能追加 / バグ修正 / リファクタリング / ドキュメント / テスト / 設定変更
- **影響範囲**: 変更されたファイル・モジュール・コンポーネント
- **主要な変更内容**: 何が追加・削除・変更されたか
- **関連するコンテキスト**: コミットメッセージ、ファイル名、関数名から意図を推測

### 3. PRタイトルの生成

変更の種類と内容に基づいて簡潔なタイトルを生成する。

```
形式例:
feat: ユーザー認証機能の追加
fix: ログイン時のセッション切れ問題を修正
refactor: データ取得ロジックをサービス層に分離
chore: CI/CDパイプラインの設定更新
```

引数でタイトルが指定されていればそれを優先する。

### 4. PR説明文の生成

以下のテンプレートに従ってPR説明文を生成する。各セクションはdiff分析に基づいて具体的な内容を記述すること。

```markdown
<!-- I want to review in Japanese. -->
## Summary
<!-- 変更の目的と背景を2〜3文で説明 -->

## 変更内容
<!-- 主要な変更点を箇条書きで記載 -->
-
-

## 変更ファイル
<!-- 重要なファイルとその変更理由 -->
| ファイル | 変更内容 |
|---------|---------|
| `path/to/file` | 説明 |

## テスト計画
<!-- 動作確認の手順やチェックリスト -->
- [ ]
- [ ]

## 注意事項 / レビュー観点
<!-- レビュアーに特に見てほしい点、懸念事項があれば記載 -->

🤖 Generated with [Claude Code](https://claude.com/claude-code)
<!-- I want to review in Japanese. -->
```

差分が小さい場合（数ファイル・数十行以内）は「変更ファイル」テーブルを省略してもよい。
変更が多い場合は「注意事項」セクションに設計上の判断や代替案を記載する。

### 5. PRの作成

```bash
gh pr create \
  --title "<生成したタイトル>" \
  --body "$(cat <<'EOF'
<生成した説明文>
EOF
)"
```

ベースブランチは基本的に `main` を使用する。`main` が存在しない場合は `master` を使用する。

### 6. 完了報告

PRのURLをユーザーに提示する。

```
✅ Pull Requestを作成しました

タイトル: <PR タイトル>
URL: <PR URL>
```

## エラーハンドリング

- **ghコマンドが未認証の場合**: `gh auth login` の実行を案内する
- **リモートにブランチがpushされていない場合**: `git push -u origin <branch>` を実行してからPRを作成する
- **mainとの差分がない場合**: ユーザーに確認してから作業を続ける
- **フォーク元リポジトリがある場合**: `--base` オプションで適切なブランチを指定する

## 注意事項

- コミットされていない変更はPRに含まれないため、必要に応じてコミットを促す
- センシティブな情報（APIキー、パスワード等）がdiffに含まれていたら警告する
- PRの説明文は日本語で生成する（リポジトリの言語設定や既存PRに合わせて調整可）
